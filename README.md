Utilizar GitHub y almacenamiento de datos de usuario
1. GitHub y repositorios
GitHub es una plataforma basada en la nube que permite a los desarrolladores almacenar, gestionar y compartir código mediante repositorios. Utiliza el sistema de control de versiones Git, lo que facilita el seguimiento de cambios en los proyectos.
Un repositorio es un espacio donde se guarda todo el código de una aplicación, incluyendo archivos, historial de cambios y versiones del proyecto.
Características principales:
•	Control de versiones 
•	Trabajo colaborativo 
•	Respaldo del código en la nube 
•	Organización de proyectos 
Importancia:
GitHub permite a los desarrolladores trabajar en equipo, evitar la pérdida de información y mantener un historial ordenado de modificaciones.

2. Almacenamiento y conservación de datos de usuario
En el desarrollo de aplicaciones web, es necesario guardar información de los usuarios para mejorar la experiencia, mantener sesiones activas y proteger datos importantes.
Estos datos pueden incluir:
•	Nombre de usuario 
•	Preferencias 
•	Estado de sesión 
•	Información de navegación 
Existen diferentes métodos para almacenar estos datos, entre los más importantes están: cookies y sesiones.
________________________________________

3. Uso de cookies
Las cookies son pequeños archivos de texto que se almacenan en el navegador del usuario cuando visita una página web.
Funciones principales:
•	Recordar datos de inicio de sesión 
•	Guardar preferencias del usuario 
•	Rastrear comportamiento en el sitio 
Características:
•	Se almacenan en el navegador 
•	Tienen fecha de expiración 
•	Son accesibles desde el cliente (JavaScript) 
Ventajas:
•	Fácil implementación 
•	Persistencia de datos 
Desventajas:
•	Menor seguridad 
•	Pueden ser eliminadas por el usuario 
________________________________________
4. Creación de sesiones
Las sesiones permiten identificar a un usuario durante su interacción con una aplicación web. A diferencia de las cookies, las sesiones se almacenan en el servidor.
Funcionamiento:
1.	El usuario inicia sesión 
2.	El servidor crea una sesión 
3.	Se asigna un identificador único (ID) 
4.	El usuario mantiene su estado mientras navega 

Funciones principales:
•	Mantener al usuario autenticado 
•	Guardar información temporal 
•	Mejorar la seguridad 
Ventajas:
•	Mayor seguridad que las cookies 
•	Los datos no se almacenan directamente en el navegador 
Desventajas:
•	Consumen recursos del servidor 
•	Expiran después de cierto tiempo 
________________________________________
5. Eliminación de sesiones
Eliminar una sesión significa cerrar la sesión del usuario y borrar los datos almacenados en el servidor.
¿Cuándo se eliminan?
•	Cuando el usuario cierra sesión 
•	Cuando la sesión expira 
•	Por motivos de seguridad 
Importancia:
•	Protege la información del usuario 
•	Evita accesos no autorizados 
•	Mantiene la integridad del sistema 
________________________________________
Conclusión
El uso de GitHub facilita la organización y control de proyectos web mediante repositorios. Por otro lado, el almacenamiento de datos de usuario mediante cookies y sesiones es fundamental para el funcionamiento de aplicaciones modernas, ya que permite mantener la información del usuario, mejorar la experiencia y garantizar la seguridad.
